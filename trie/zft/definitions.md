
### Weak prefix search

 
Для заданного префикс-свободного множества строк $S\subseteq2^{*}$ проблема слабого
префиксного поиска требует, чтобы по префиксу $p$ некоторой строки в $S$ возвращался диапазон строк из $S$, имеющих $p$ в качестве префикса;
этот набор возвращается в виде интервала целых чисел, которые являются рангами (в лексикографическом порядке) строк в $S$, имеющих $p$ в
качестве префикса.


### Exit Node

Узел $\alpha$ является узлом выхода для строки $z$ тогда и только тогда, когда выполняются два условия:
1) Имя узла ($n_\alpha$) является префиксом строки $z$
2) Либо расширение узла ($e_\alpha$) в точности равно $z$, либо $e_\alpha$ не является префиксом строки $z$3

В контексте слабого префиксного поиска _(а имеено реализации Hollow-ZFT)_ (где $p$ — префикс какой-то строки из набора $S$).  
**Искомая строка $z$ всегда короче или равна $e_\alpha$.**  
Если строка $z$ длиннее, чем $e_\alpha$, и при этом $e_\alpha$ является префиксом $z$, то согласно определению, 
$\alpha$ уже не может быть узлом выхода.
В таком случае «поиск» прошел бы глубже в дерево к одному из потомков узла $\alpha$.
Таким образом, для узла выхода всегда справедливо $|n_\alpha| \le |z| [cite_start]\le |e_\alpha|$.

Так же хочу заметить, что в Hollow-ZFT $n_{\alpha}$ **всегда** является pseudohandl-ом.
